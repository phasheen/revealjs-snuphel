Development

Setting Up Your Environment

  • Create a NodeJS Workspace in Coder
  • Clone the template repository
  • Install dependencies with npm install
  • Start the dev server with npm start
  • Edit files in your preferred code editor
  • Changes appear instantly in browser

# Clone the repository
git clone https://github.com/phasheen/revealjs-snuphel

# Navigate to the directory
cd revealjs-snuphel

# Install dependencies
npm install

# Start the development server
npm start
Setting up your development environment is straightforward. The template uses npm for package management and includes a development server that automatically refreshes as you make changes.

Basic Slide Structure

  • Each slide is a <section> element
  • Vertical slides use nested sections
  • Use HTML or Markdown for content
  • Add notes with <aside class="notes">
<section>
  <h2>Slide Title</h2>
  <ul>
    <li>Bullet point 1</li>
    <li>Bullet point 2</li>
  </ul>
  <aside class="notes">
    Speaker notes here
  </aside>
</section>
The basic structure of a Reveal.js presentation is simple: each slide is a section element, and you can create vertical slides by nesting sections within each other.

Deployment with GitHub Pages

Why GitHub Pages?

  • Free hosting for your presentations
  • Integrated with Git workflows
  • Automatic builds with GitHub Actions
  • Custom domains with SSL certificates
  • Easy collaboration with teammates

GitHub Pages deployment overview

https://pages.github.com/
https://docs.github.com/en/pages
GitHub Pages offers an excellent free solution for hosting academic presentations, with automatic deployments through GitHub Actions, version control, and easy sharing.

GitHub Pages Setup

  1. Create a new GitHub repository
  2. Push your presentation files to the repository
  3. Go to repository Settings â†’ Pages
  4. Set source to "GitHub Actions"
  5. The included workflow file will handle deployment automatically

# Initialize git in your project folder
git init

# Add all files
git add .

# Commit the changes
git commit -m "Initial commit of my presentation"

# Add remote repository
git remote add origin https://github.com/username/my-presentation.git

# Push to GitHub
git push -u origin main
Setting up your presentation in GitHub is straightforward. The template already includes a GitHub Actions workflow file that will automatically deploy your presentation to GitHub Pages whenever you push changes to the repository.

GitHub Actions Workflow

  • Included in the template as .github/workflows/github-pages.yml
  • Automatically deploys on push to main branch
  • No configuration needed
  • Fast deployment (usually less than a minute)

name: Deploy to GitHub Pages

on:
  push:
    branches:
      - main

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
      
      - name: Setup Pages
        uses: actions/configure-pages@v4
      
      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: '.'
      
      - name: Deploy to GitHub Pages
        uses: actions/deploy-pages@v4
The GitHub Actions workflow file is already configured to deploy your presentation to GitHub Pages. It will automatically run whenever you push changes to the main branch of your repository.
